# 🌾 RL Irrigation Module

Reinforcement Learning-based irrigation optimization for the Smart Irrigation System.
Designed as a **plug-and-play** module — does NOT modify any existing API or route.

---

## Folder Structure

```
rl_irrigation/
├── __init__.py          ← public package exports
├── environment.py       ← Gymnasium IrrigationEnv
├── train.py             ← PPO training script (50k steps)
├── predict.py           ← get_irrigation_action() — call this from rag_api.py
├── requirements.txt     ← dependencies
├── README.md            ← this file
└── models/
    ├── ppo_irrigation.zip   ← trained model (generated by train.py)
    └── best_model.zip       ← best checkpoint (auto-saved by EvalCallback)
```

---

## State Space

| Feature | Range | Description |
|---------|-------|-------------|
| `soil_moisture` | 0 – 100 | Soil moisture % |
| `temperature` | 0 – 50 | Air temperature °C |
| `humidity` | 0 – 100 | Relative humidity % |
| `rain` | 0 / 1 | Rain detected (binary) |
| `crop_stage` | 0 – 4 | Crop growth stage |

## Action Space → Litres

| Action | Litres |
|--------|--------|
| 0 | 0 L (no irrigation) |
| 1 | 10 L |
| 2 | 20 L |
| 3 | 30 L |

## Reward Function

| Condition | Reward |
|-----------|--------|
| Soil in [40–70]% | +2.0 |
| Water usage cost | −0.1 × litres |
| Soil > 80% (over-irrigation) | −3.0 |
| Soil < 20% (crop stress) | −2.0 |
| Irrigating during rain | −1.5 |
| Near-harvest crop stage bonus | +0.1 × stage |

## FastAPI Integration (already done in rag_api.py)

```python
# At module level (startup)
from rl_irrigation.predict import load_rl_model, get_irrigation_action
rl_model = load_rl_model()

# Endpoint
@app.post("/rl/irrigation")
def rl_irrigation(req: RLRequest):
    return get_irrigation_action(
        req.soil_moisture, req.temperature,
        req.humidity, req.rain, req.crop_stage,
        model=rl_model,
    )
```

## Node.js Integration (already done in rl.service.js + server.js)

The Node.js backend calls `POST /rl/irrigation` after each MQTT sensor message
and emits the result as a `rl-action` Socket.IO event to the frontend.

```js
socket.on("rl-action", (data) => {
  console.log(`RL recommends: ${data.litres}L — ${data.reasoning}`);
});
```

## Notes for Hackathon

- **No model file?** The system automatically falls back to a rule-based heuristic, so the endpoint works immediately even before training.
- **Re-train** to improve accuracy: just run `train.py` again.
- The model is **stateless** — each call is independent, safe for concurrent requests.
